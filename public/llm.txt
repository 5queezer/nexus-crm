# Job Tracker — LLM Access Guide

## What this app is
A personal job application tracker. Supports CRUD for applications, contacts, and documents.
Multi-tenant: each user only sees their own data. One admin token bypasses tenant scoping.

## Authentication

### User session (Google OAuth)
- Login: GET /login → redirects to Google OAuth
- Session managed by better-auth (httpOnly cookies)

### Admin API token (Bearer)
- Header: `Authorization: Bearer $ADMIN_API_TOKEN`
- The token value is set via environment variable on the server — never hardcoded
- Admin token bypasses userId filtering (sees all tenants)
- Store the token in your secrets manager; never commit it to version control

### Share token (read-only public)
- Query param: `/share?token=$SHARE_TOKEN`
- Read-only view of all applications and documents
- Token set via environment variable — never hardcoded

## REST API

### Applications
| Method | Path | Auth | Description |
|--------|------|------|-------------|
| GET | /api/applications | session or admin | List all (filtered by userId unless admin) |
| POST | /api/applications | session or admin | Create application |
| GET | /api/applications/:id | session or admin | Get single |
| PATCH | /api/applications/:id | session or admin | Update |
| DELETE | /api/applications/:id | session or admin | Delete |

### Contacts (per application)
| Method | Path | Auth | Description |
|--------|------|------|-------------|
| GET | /api/applications/:id/contacts | session or admin | List contacts |
| POST | /api/applications/:id/contacts | session or admin | Create contact |
| PATCH | /api/applications/:id/contacts/:contactId | session or admin | Update contact |
| DELETE | /api/applications/:id/contacts/:contactId | session or admin | Delete contact |

### Documents
| Method | Path | Auth | Description |
|--------|------|------|-------------|
| GET | /api/documents | session or admin | List documents |
| POST | /api/documents | session or admin | Upload (multipart/form-data, field: file) |
| DELETE | /api/documents/:id | session or admin | Delete |
| GET | /api/documents/:id/file | session, admin, or share token | Download file |

---

## Application status values
`draft` | `applied` | `waiting` | `interview` | `offer` | `rejected` | `ghost`

## Example: create application via admin token
```
POST /api/applications
Authorization: Bearer <ADMIN_API_TOKEN from env>
Content-Type: application/json

{
  "company": "Acme Corp",
  "role": "Senior Engineer",
  "status": "applied",
  "appliedAt": "2026-03-06",
  "url": "https://acme.com/jobs/123",
  "notes": "Referral from Alice"
}
```

## Security notes
- ADMIN_API_TOKEN and SHARE_TOKEN are server-side env vars only
- NEXT_PUBLIC_* env vars do NOT contain any tokens
- Rate limiting: 60 req/min per IP on API routes
- All inputs validated; max upload size 15MB
- Row Level Security: userId enforced at query level via lib/tenant.ts
